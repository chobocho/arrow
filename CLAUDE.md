# 프로젝트 가이드라인:

## 1. 개요
- **목적:** HTML5 CANVAS 기반의 화살표만 지원하는 변형 마인드 맵 
- **엔진/언어:** Typescipt
- **핵심 목표:** 동작 가능한 웹앱
- 참고소스를 HTML5 CANVAS + Typescript 로 포팅

## 2. 작업 규칙
- 모든 변경은: 설계 → 테스트 작성 → 구현 → 검증 순서로 진행
- 작업 완료 후 history.html에 한글로 이력 기록 (최신 항목을 첫 `<h2>` 위에 추가)
- TODO 항목은 todo.html에 한글로 관리 (완료시 끝에 ✅)
- readme.md는 한글로 유지
- 한 번에 하나의 이슈만 해결하고 테스트 통과 확인 후 git commit 후 다음 아이템으로
- 관리 에이전트가 개발, 검증, 디자인 에이전트 관리
- 작업 완료 후 github 푸시

## 3. 개발 주의 사항
- 실행시 외부 라이이브러리 참조가 있으면 안됨
- HTML5 Canvas기반으로 해주세요
- python -m http.server 8001 로 동작 가능해야 함

## 4. 이미지, 사운드 리소스
* 이모를 최대한 사용
* 인터넷 무료 이미지 리소스 사용시 출처 표기
* 인터넷 무료 오디오 리소스 사용시 출처 표기 
* 오디오 리소스는 base64로 바꾸어 js에 포함 시킬 것

## 5. 코딩 규칙 (Coding Rules)
- 모든 코드는 모듈화하여 작성 (scene, player, enemy, ui 분리)
- 변수명은 camelCase, 클래스명은 PascalCase 사용
- 주석은 코드의 의도를 명확히 설명할 것

## 6. 저장 메커니즘 (DB and Store)
- **DB**: 브라우저 indexDB 사용
- json 형식을 데이터 파일 저장 및 볼러오기 가능
- 내부 db을 통하여 여러 작업물을 저장 및 불러 오기 가능
- 모든 DB내용을 하나의 json으로 다운로드 및 등록 가능
- 내부 db에 저장된 작업물의 삭제, 이름 변경, 불러오기 가능
- 이어하기 지원

## 6. 상세 지침 (Directives)
- 코드를 수정하기 전에 `test` 폴더의 기존 로직을 확인하라.
- 성능에 미치는 영향을 고려하여 최적화된 코드를 작성하라.
- DB손상, 오류시에도 동작해야함
- PC 브라우저 동작 가능 해야함
- 모바일 터치 UI지원
- 폴더7 펼침화면, 열린 화면 모두 지원
- 폴더7 펼침 화면에서 고해상도로 보이도록 할 것
- 데이터 형식은 json형식으로 수정과 추가가 쉬워야함
- 사용자의 진행 상항은 Browser의 IndexedDB의 디비에 저장해서 이어 하기 가능해야 함

## 7. 목표
- 화살표만 지원하는 변형 마인드 맵
- 가운데 주제를 넣을 수 있다
- Click & Drag 로 화살표를 넣을 수 있다 
- 화살표는 터치를 통하여 크기와 방향을 바꿀 수 있다.
- 언어는 한글과 영어를 지원한다
- 글자(단어/문장) 입력 or 화살표 입력 두가지만 지원한다.
- 입력된 글자도 하나의 객체로 터치로 이동 및 크기 조정이 가능하다
- 폰트는 브라우저가 지원하는 기본 폰트만 사용한다 
- 캔버스는 최대 가로 4096 x 4096 px
- 캔버스는 두 손가락을 확대 축소가 가능하다
- 캔버스를 png로 저장 할 수 있다
